RunAsRob Übersicht

Starte Programm als Admin, mit dem Systemkonto, als Dienst oder anderer Benutzer

RunAsRob ist nicht nur ein Runas mit Passwort Befehl, sondern bietet vier komplett unterschiedliche Anmeldeoptionen, mit welcher man Software als Administrator mit Systemrechte starten kann.

Ein Standardbenutzer kann temporär in die Gruppe der lokalen Administratoren geschoben werden, nur für das autorisierte Programm
oder RunAsRob startet die erlaubte Anwendung mit dem Systemkonto, dieses Konto hat sogar mehr lokale Rechte als ein Administrator.
Eine weitere variante kann Anwendungen als Dienst starten, um das Programm nach dem Booten auszuführen, bevor sich ein Benutzer am System anmeldet.
Die vierte Option startet Software unter anderem Benutzer wie bei dem Befehl runas, allerdings mit der Möglichkeit das Passwort verschlüsselt zu übergeben.

Ein anderer Vorteil von RunAsRob ist, die Möglichkeit einen ganzen Ordner zu autorisieren, von woraus sie ein Programm mit Administratorrechte starten können.

Anwendungsmöglichkeiten

  • Autorisieren Sie Programmdateien oder Ordner in RunAsRob, die von einem eingeschränkten Benutzer genutzt werden können um Software als Administrator starten zu lassen.
  • Sie können Software, Updates, Patches, Treiber,... sehr einfach verteilen, indem sie die Installationsdateien auf eine Netzwerkfreigabe kopieren, die in RunAsRob autorisiert wurde.
  • Geben sie nur einem bestimmten Programm Administratorrechte, statt einem ganzem Benutzerkonto.
  • Delegieren sie administrative Arbeiten an Benutzer, indem sie entsprechende Aufgabe erlauben als Administrator zu starten.
  • Verwalten sie die erlaubten Anwendungen zentral in einer Domain über Gruppenrichtlinien, Organisationseinheiten und Verzeichnisrechte. RunAsRob harmoniert damit sehr gut.
  • Umgehen der UAC Abfrage für ein Script oder ein Programm.
  • Eine Anwendungen oder ein Script kann unter dem System Konto starten, welches mehr Systemrechte hat als ein Administrator Konto.
  • Starte ein Programm als Admin mit dem eigenem Profil und eigenen Einstellungen des Benutzers.
    In anderen Runas Tools ist es nur möglich Programme unter einem anderen Konto als Administrator zu starten.
  • Wildcard Asterisk * und Umgebungsvariablen wie %windir% können für das autorisieren von relativen Ordner - und Programmpfaden verwendet werden.
  • Sie benötigen keine Anmeldedaten von einem Administrator Konto in einer verschlüsselten Datei oder sonst irgendwo.
    Die Autorisierung und Anmeldeinformationen können über Registry, Verzeichnisrechte und einem Service kontrolliert werden.
  • Starte ein Programm als Dienst mit Systemrechte bereits nach dem Booten des Rechners, ob ein Benutzer angemeldet ist oder nicht.
  • RunAsRob ist effectiv und einfach zu verwenden bei lokalen Rechnern, in kleinen Netzwerken oder in einer großen Domainumgebung, weil das Design klar und übersichtlich ist.
